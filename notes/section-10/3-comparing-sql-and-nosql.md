# SQL vs NoSQL

## SQL

- Data uses Schema
- Relations
- Data is disctributes accross multiaple tables
- Horizontal scaling is difficult / imposible; Verical scaling is posible
- Limitaions for lots of (thousand) read & write queris perSecond

## No SQL

- Schema-less
- No (or very few) Relations
- Data is typically merged / nested in a few collections
- Both horizontal and vertical scalling is possible
- Great performance fo mass read & write request
