# Adding a New Product to the Cart

## Overview

When a user clicks "Add to Cart", this handler fetches their cart, checks if the product already exists in it, then either increments the quantity or adds it fresh. The quantity update logic is stubbed here and completed in the next section.

---

## Handler (`shop.js`)

```js
exports.postCart = (req, res) => {
  const id = req.body.productId; // Product ID sent from the form
  let fetchedCart; // Hoisted outside .then() so all steps can access it

  req.user
    .getCart() // Step 1: Get this user's cart
    .then((cart) => {
      fetchedCart = cart; // Save cart reference for later steps

      // Step 2: Check if this product is already in the cart
      return cart.getProducts({ where: { id } });
    })
    .then((products) => {
      // products is an array — grab the first match if it exists
      const product = products.length > 0 ? products[0] : null;

      let newQuantity = 1; // Default quantity for a brand-new cart item

      if (product) {
        // Product already exists in cart — quantity update handled in next section
      }

      // Step 3: Fetch the full product from the DB, then add it to the cart
      return Product.findByPk(id)
        .then((product) => {
          // addProduct() is a Sequelize magic method for belongsToMany
          // `through` lets you set extra fields on the junction table (cart_items)
          return fetchedCart.addProduct(product, {
            through: { quantity: newQuantity },
          });
        })
        .catch((error) => console.log(error));
    })
    .then(() => {
      res.redirect("/cart"); // Step 4: Redirect back to cart page
    })
    .catch((error) => console.log(error));
};
```

---

## How It Works

### Why `fetchedCart` is Declared Outside `.then()`

Promise chains don't share scope between steps. By declaring `fetchedCart` with `let` before the chain starts, every `.then()` block can read and write to it.

```
let fetchedCart;
  └── .then(cart => { fetchedCart = cart })   ← writes
  └── .then(() => { fetchedCart.addProduct }) ← reads
```

### Why Two Separate Fetches?

| Fetch                                 | Method         | Purpose                                               |
| ------------------------------------- | -------------- | ----------------------------------------------------- |
| `cart.getProducts({ where: { id } })` | Magic method   | Check if product is _already in the cart_             |
| `Product.findByPk(id)`                | Standard query | Get the full product record to pass to `addProduct()` |

`getProducts()` filters by what's in `cart_items`. `findByPk()` queries the `products` table directly. Both are needed.

### `addProduct()` and the `through` Option

```js
fetchedCart.addProduct(product, { through: { quantity: newQuantity } });
```

`addProduct()` is auto-generated by Sequelize for `belongsToMany` relationships. The `through` option lets you write extra columns directly into the junction table (`cart_items`) at insert time — in this case, `quantity`.

### Full Step Flow

```
POST /cart (productId in body)
  └── req.user.getCart()
        └── cart.getProducts({ where: { id } })   ← is it already in the cart?
              └── Product.findByPk(id)             ← get full product record
                    └── cart.addProduct(product, { through: { quantity } })
                          └── res.redirect("/cart")
```

---

## Key Notes

- **`addProduct()`** is a Sequelize magic method created automatically from `Cart.belongsToMany(Product, { through: CartItem })`
- **`through: { quantity }`** writes directly to the `cart_items` junction table — this is how extra columns on a join table are set
- The `if (product)` branch is intentionally incomplete here — quantity incrementing is covered in the next section
- Consider `async/await` to flatten the nested `.then()` chains as this logic grows
