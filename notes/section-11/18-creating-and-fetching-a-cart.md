# Creating and Fetching a Cart

## Overview

This document covers how to attach a user to every incoming request, then use Sequelize's auto-generated methods to fetch that user's cart and its products.

---

## Step 1 — Attach User to Every Request (`app.js`)

```js
// Middleware: runs before every route handler
// Finds User with id=1 and attaches them to req.user
app.use((req, _, next) => {
  User.findByPk(1) // Fetch user by primary key
    .then((user) => {
      req.user = user; // Attach the Sequelize instance to the request
      next(); // Continue to the next middleware/route
    })
    .catch((error) => console.log(error));
});
```

> **Why a Sequelize instance?** Storing the full Sequelize model instance (not just plain data) on `req.user` gives you access to all of Sequelize's auto-generated association methods like `getCart()`, `getProducts()`, etc. in your route handlers.

> ⚠️ **Note:** Hardcoding `findByPk(1)` is only for development. In production, the user ID comes from session/auth middleware.

---

## Step 2 — Fetch the Cart and Its Products

```js
exports.getCart = (req, res) => {
  req.user
    .getCart() // Sequelize magic method: fetches the cart belonging to this user
    .then((cart) => {
      /*
        cart.dataValues looks like:
        {
          id: 1,
          createdAt: 2026-03-13T04:22:19.965Z,
          updatedAt: 2026-03-13T04:22:19.965Z,
          userId: 1          // foreign key linking back to the user
        }
      */

      return cart
        .getProducts() // Sequelize magic method: fetches all products in this cart (via cart_items)
        .then((products) => {
          res.render("shop/cart", {
            pageTitle: "Cart - Shop",
            path: "/cart",
            products, // Array of Product instances, each including quantity from cart_items
          });
        })
        .catch((error) => console.log(error));
    })
    .catch((error) => console.log(error));
};
```

---

## How It Works

Sequelize automatically generates helper methods based on your associations. When you define:

```js
User.hasOne(Cart);
Cart.belongsToMany(Product, { through: CartItem });
```

Sequelize creates these methods for free:

| Method          | Available On    | What It Does                                 |
| --------------- | --------------- | -------------------------------------------- |
| `getCart()`     | `user` instance | Fetches the cart row where `userId` matches  |
| `getProducts()` | `cart` instance | Fetches all products linked via `cart_items` |

### Request Flow

```
Incoming Request
  └── app.use middleware
        └── User.findByPk(1)  →  attaches user instance to req.user
              └── Route Handler: getCart
                    └── req.user.getCart()
                          └── cart.getProducts()
                                └── res.render("shop/cart", { products })
```

---

## Key Notes

- **`getCart()` / `getProducts()`** are auto-generated by Sequelize — you never write these manually
- The method name is derived from the model name: `hasOne(Cart)` → `getCart()`, `belongsToMany(Product)` → `getProducts()`
- `getProducts()` automatically joins through `cart_items`, so each product can also carry its `quantity`
- Nested `.then()` chains can be flattened with `async/await` to improve readability as the app grows
